# Contributing

Thanks for contributing to `intent_ledger`.

## Development

```sh
mix deps.get
mix test
mix quality
```

Before opening a pull request, make sure formatting, compile warnings, tests,
Credo, Dialyzer, and Doctor all pass locally.

## Pull Requests

- Keep changes scoped to one concern.
- Add tests for behaviour changes.
- Update guides when public APIs or lifecycle semantics change.
- Use conventional commit messages when possible.
